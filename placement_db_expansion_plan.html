New tables to add
TABLE
placement_rounds
Tracks each round of an interview process per offer — technical, HR, group discussion, case study. Has round_number, type, scheduled_at, duration_minutes, max_students.
1-to-many with offers · ENUM round type · CHECK constraints
TABLE
round_results
Records each student's result per round — pass/fail/pending, score out of 100, evaluator remarks. Links application_id + round_id with a composite PK.
Composite PK · many-to-many bridge · DECIMAL score
TABLE
feedback
Companies leave feedback on students after selection/rejection. Rating 1–5, comments, anonymous flag. Useful for analytics and student improvement.
FK to applications · CHECK rating 1–5 · BOOLEAN
TABLE
documents
Stores multiple documents per student — resume versions, NOC, offer letter, ID proof. Has doc_type ENUM, file_path, verified BOOLEAN, uploaded_at.
1-to-many with students · ENUM doc_type · versioning
TABLE
placement_stats
Materialized summary table updated by a trigger — total placed, avg stipend, max stipend per branch per year. Used for dashboard analytics without expensive live aggregation.
Denormalized analytics · updated by trigger · GROUP BY source
TABLE
blacklist
Admin can blacklist a student from applying (e.g. misconduct). Has student_id, reason, blacklisted_by, active BOOLEAN, expires_at DATE.
Checked by BEFORE INSERT trigger on applications
New triggers
TRIGGER
trg_block_blacklisted_student
BEFORE INSERT on applications — checks blacklist table. If student is active-blacklisted, SIGNALs an error and blocks the insert.
BEFORE INSERT · SIGNAL SQLSTATE · subquery in trigger
TRIGGER
trg_auto_reject_on_fail
AFTER INSERT on round_results — if result is 'fail', automatically sets the application status to 'rejected' and inserts a notification.
AFTER INSERT · cascading UPDATE · cross-table logic
TRIGGER
trg_update_placement_stats
AFTER UPDATE on applications — when status changes to 'selected', refreshes the placement_stats row for that student's branch and year.
AFTER UPDATE · conditional logic · aggregate refresh
TRIGGER
trg_log_offer_changes
AFTER UPDATE on offers — logs old vs new values (stipend, deadline, status) into audit_log as JSON. Full change history for every offer.
AFTER UPDATE · JSON_OBJECT() · audit trail
Stored functions (return a single value, usable in SELECT)
FUNCTION
fn_student_eligibility(student_id, offer_id)
Returns 'eligible', 'low_cgpa', 'too_many_backlogs', or 'wrong_branch'. Use it in WHERE clauses or SELECT to explain why a student can't apply.
RETURNS VARCHAR · IF/ELSEIF · reusable in queries
FUNCTION
fn_days_to_deadline(offer_id)
Returns integer days remaining until offer deadline. Negative means expired. Use in SELECT to show countdown: SELECT title, fn_days_to_deadline(id) AS days_left FROM offers.
RETURNS INT · DATEDIFF() · CURDATE()
FUNCTION
fn_placement_rate(branch_code)
Returns the placement rate as a percentage for a given branch — (selected students / total students) × 100. Used in the stats dashboard.
RETURNS DECIMAL · COUNT with condition · percentage calc
Stored procedures (complex multi-step logic)
PROCEDURE
sp_advance_to_next_round(offer_id, round_number)
Takes all students who passed round N of an offer and auto-creates their round_results entry for round N+1 with status 'pending'. Admin calls this to advance a batch.
Cursor · LOOP · INSERT from SELECT
PROCEDURE
sp_bulk_reject(offer_id)
Sets all 'pending' applications for an offer to 'rejected' and inserts a notification for each student in a single transaction. Useful when an offer is withdrawn.
Transaction · ROLLBACK on error · bulk UPDATE + INSERT
PROCEDURE
sp_generate_report(branch_code, year)
Outputs a full placement report for a branch/year — total students, applied, shortlisted, selected, avg CGPA of placed students, top company by selections.
Multiple result sets · aggregation · GROUP BY · HAVING
New views
VIEW
vw_placement_leaderboard
Students ranked by CGPA within each branch using RANK() OVER (PARTITION BY branch). Shows name, branch, cgpa, branch_rank, overall_rank, placement status.
Window functions · RANK() · PARTITION BY
VIEW
vw_offer_pipeline
Per-offer summary with applicant counts at each stage — pending, shortlisted, in rounds, selected, rejected. Like a Kanban column count per offer.
CASE WHEN aggregation · COUNT with filter · LEFT JOIN
VIEW
vw_company_performance
Per-company stats — total offers, total applications received, conversion rate (selected/applied %), average stipend offered, highest stipend.
Multi-table join · ratio calculation · GROUP BY company
